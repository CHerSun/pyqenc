## Introduction

This feature extends the existing video metrics analysis capability to support unified visualization of PSNR, SSIM, and VMAF metrics. Currently, only PSNR has a dedicated parsing and plotting module. This enhancement will create a reusable framework for parsing all three metric types, computing statistics, and generating comparative visualizations with dual Y-axes to accommodate different metric scales.

## Glossary

- **PSNR (Peak Signal-to-Noise Ratio)**: A video quality metric measured in decibels (dB). Higher values indicate better quality. Values can be infinite for identical frames. Upper cutoff of 100 dB is used for visualization.
- **SSIM (Structural Similarity Index)**: A video quality metric ranging from 0.0 to 1.0 (or 0% to 100%), where 1.0 represents lossless quality.
- **VMAF (Video Multimethod Assessment Fusion)**: A perceptual video quality metric ranging from 0.0 to 100.0, where 100.0 represents lossless quality.
- **Metric Parser**: A component that reads metric log files and extracts frame-by-frame quality measurements.
- **Unified Visualization**: A graphing system that displays multiple metrics on a single plot with appropriate scaling.
- **Statistics Module**: A component that computes quantile-based statistics (min, percentiles, max, standard deviation) for quality metrics.
- **Dual Y-Axis Plot**: A graph with two Y-axes to accommodate metrics with different scales (PSNR in dB vs SSIM/VMAF in percentage).

## Requirements

### Requirement 1

**User Story:** As a video quality analyst, I want to parse SSIM log files generated by FFmpeg, so that I can analyze SSIM quality measurements frame-by-frame.

#### Acceptance Criteria

1. WHEN the Metric Parser receives an SSIM log file path, THE Metric Parser SHALL extract frame numbers and SSIM values from each line
2. WHEN the Metric Parser processes SSIM data, THE Metric Parser SHALL compute actual frame numbers using the provided sampling factor
3. WHEN the Metric Parser encounters an invalid SSIM log file, THE Metric Parser SHALL raise a ValueError with a descriptive message
4. THE Metric Parser SHALL return SSIM data as a pandas DataFrame with frame numbers as the index

### Requirement 2

**User Story:** As a video quality analyst, I want to parse VMAF JSON log files generated by FFmpeg, so that I can analyze VMAF quality measurements frame-by-frame.

#### Acceptance Criteria

1. WHEN the Metric Parser receives a VMAF JSON file path, THE Metric Parser SHALL extract frame numbers and VMAF scores from the JSON structure
2. WHEN the Metric Parser processes VMAF data, THE Metric Parser SHALL compute actual frame numbers using the provided sampling factor
3. WHEN the Metric Parser encounters an invalid VMAF JSON file, THE Metric Parser SHALL raise a ValueError with a descriptive message
4. THE Metric Parser SHALL return VMAF data as a pandas DataFrame with frame numbers as the index

### Requirement 3

**User Story:** As a video quality analyst, I want to compute statistics for SSIM and VMAF metrics using the same statistical measures as PSNR, so that I can compare quality distributions across different metrics.

#### Acceptance Criteria

1. THE Statistics Module SHALL compute quantiles at levels 0%, 5%, 10%, 25%, 50%, 75%, 90%, 95%, and 100% for any metric series
2. THE Statistics Module SHALL compute standard deviation with configurable upper and lower cutoff values to exclude outliers
3. WHEN computing standard deviation, THE Statistics Module SHALL exclude infinite and NaN values from the calculation
4. THE Statistics Module SHALL return statistics as a dictionary with keys: min, p5, p10, p25, p50, p75, p90, p95, max, std

### Requirement 4

**User Story:** As a video quality analyst, I want to visualize all three metrics (PSNR, SSIM, VMAF) on a single graph with dual Y-axes, so that I can compare quality trends across different measurement approaches.

#### Acceptance Criteria

1. THE Unified Visualization SHALL display PSNR values on the left Y-axis with a range of 0-103 dB
2. THE Unified Visualization SHALL display SSIM and VMAF values on the right Y-axis with a range of 0-100%
3. WHEN plotting multiple metrics, THE Unified Visualization SHALL use distinct colors and line styles for each metric
4. THE Unified Visualization SHALL include a legend identifying each metric with its corresponding line
5. WHEN PSNR contains infinite values, THE Unified Visualization SHALL mark these frames with special markers at the 100 dB position

### Requirement 5

**User Story:** As a video quality analyst, I want to see individual statistics bars for each metric type, so that I can quickly assess the quality distribution of each measurement approach.

#### Acceptance Criteria

1. THE Unified Visualization SHALL create separate horizontal bar charts for each available metric's statistics
2. THE Unified Visualization SHALL display percentile bars (Min, 5%, 10%, 25%, 50%, 75%, 90%, 95%, Max) for each metric
3. THE Unified Visualization SHALL label each bar with its numeric value and appropriate unit (dB for PSNR, % for SSIM/VMAF)
4. THE Unified Visualization SHALL use consistent color schemes within each metric's bar chart

### Requirement 6

**User Story:** As a video quality analyst, I want to see summary statistics for each metric in text boxes on the graph, so that I can quickly understand key quality indicators without examining the full data.

#### Acceptance Criteria

1. THE Unified Visualization SHALL display a summary text box for each available metric
2. THE Unified Visualization SHALL include frame count, sampling factor, min, median, max, and standard deviation in each summary
3. WHEN PSNR data contains infinite values, THE Unified Visualization SHALL include the count of infinite PSNR frames in the PSNR summary
4. THE Unified Visualization SHALL format numeric values with appropriate precision (1 decimal place for quality values)

### Requirement 7

**User Story:** As a developer, I want to reuse common parsing, statistics, and plotting code across all three metrics, so that the codebase remains maintainable and consistent.

#### Acceptance Criteria

1. THE Metric Parser SHALL provide a unified interface that accepts metric type and file path parameters
2. THE Statistics Module SHALL operate on generic pandas Series without metric-specific logic
3. THE Unified Visualization SHALL accept multiple DataFrames and automatically configure axes based on available metrics
4. THE system SHALL minimize code duplication by extracting common functionality into shared functions

### Requirement 8

**User Story:** As a video quality analyst, I want to run the unified metrics visualization from the command line with multiple log files, so that I can generate comparative plots efficiently.

#### Acceptance Criteria

1. THE system SHALL accept command-line arguments for PSNR log file path, SSIM log file path, and VMAF JSON file path
2. THE system SHALL accept a sampling factor parameter that applies to all metrics
3. WHEN a metric log file is not provided, THE system SHALL generate visualization with only the available metrics
4. THE system SHALL save the unified plot to a file path derived from the provided log file names
5. THE system SHALL save individual statistics for each metric to separate text files

### Requirement 9

**User Story:** As a developer integrating metrics into the main encoding pipeline, I want a programmatic API that returns quality statistics for video chunks, so that I can evaluate encoding quality and adjust parameters in the scene-based encoding workflow.

#### Acceptance Criteria

1. THE system SHALL provide a function that accepts paths to PSNR, SSIM, and VMAF metric log files for a video chunk
2. THE system SHALL return a structured dictionary containing min, median, max, and standard deviation for each available metric
3. WHEN a metric file path is not provided or is None, THE system SHALL omit that metric from the returned statistics
4. THE system SHALL compute statistics without requiring full DataFrame manipulation by the caller
5. THE returned statistics SHALL be suitable for programmatic decision-making in an encoding quality feedback loop
6. THE system SHALL generate a unified visualization plot file during the API call
7. THE system SHALL write individual statistics text files for each available metric during the API call
